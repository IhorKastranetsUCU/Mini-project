# Звіт: Модуль "Sorting"

У цьому блоці курсу я повторив основні методи сортування та їхню ефективність.

## 1. Теоретична база
Я розрізняю алгоритми за їхньою часовою складністю:
* **Квадратична складність $O(n^2)$:** такі методи як **Bubble Sort** та **Insertion Sort**.
* **Логарифмічна складність $O(n \log n)$:** ефективні алгоритми **Merge Sort** та **Quick Sort**.
* **Лінійна складність $O(n)$:** алгоритм **Counting Sort**.

### Особливості Counting Sort
Раніше я вже вивчав алгоритми та підготував детальний опис:
* **[Документація: Counting Sort](counting_sort.md)**

Головним недоліком цього методу є вразливість до великих значень (діапазону даних), адже він використовує багато додаткової памʼяті. Наприклад, якщо у нас є всього два елементи — $0$ та $1,000,000$, алгоритм виконає близько $10^6$ операцій, тоді як **Bubble Sort** впорається всього за кілька кроків. Проте для списків, де варіабельність значень невелика, це дуже хороше рішення. Також важливо пам'ятати, що він працює лише з цілими числами.
