## 1. Теоретична база
Хеш-таблиця – це спосіб організації даних, де одному унікальному ключу відповідає певне значення.
* **Принцип роботи:** Під капотом хеш-таблиця – це масив. Ключі проходять через спеціальну **хеш-функцію**, яка обчислює індекс (номер), за яким значення зберігатиметься в масиві.
* **Швидкість:** Головна перевага цієї структури в тому, що додавання, видалення та доступ до даних відбувається за **$O(1)$** (в середньому), а не за $O(n)$, як у звичайних списках.
* **Реалізація в Python:** Дана структура реалізована як вбудований тип даних `dict` (словник).

### Колізії та їх вирішення
Недоліком є те, що в якийсь момент може відбутись **колізія**. Це ситуація, коли двом різним ключам хеш-функція призначає один і той самий індекс. 

---

## 2. Робота зі словниками в Python
В словник можна записувати дані різних форматів: стрічки, числа, масиви та навіть інші словники.

**Основний синтаксис:**
```python
# Створення словника
my_dict = {"name": "Ihor", "course": "DSA"}

# Додавання або оновлення значення
my_dict["city"] = "Lviv" 

# Видалення ключа
del my_dict["course"]

# Перезапис значення
my_dict["name"] = "Ihor Kastranets"
```
